# Engram Roadmap

## Mission

Memory belongs to the user.
Engram is the guardian, not the owner.
Built to make the world better, not to be acquired.

---

## Phase 1 — Foundation (done)

- ✅ Memory scoring — irreplaceability + connectivity
- ✅ Graph-based memory connections
- ✅ Causal chain detection
- ✅ Intelligent forgetting — 0.94 quality
- ✅ FAISS vector search
- ✅ Embedding cache — 2669x faster
- ✅ Production API — live on Railway
- ✅ Security — bcrypt, rate limiting, sanitization
- ✅ 51 tests passing
- ✅ LoCoMo benchmark

---

## Phase 2 — Better Brain (next)

- ⬜ Conversation-aware irreplaceability detection
- ⬜ Significance scoring — detect moments of decision, change, surprise, emotion
- ⬜ Reconstructive retrieval — follow connections to rebuild full context
- ⬜ Memory consolidation — merge related memories into stronger unified knowledge

---

## Phase 3 — True Brain Model

- ✅ Sleep consolidation — strengthen important memories, fade weak ones,
     create missing links, merge redundant fragments
- ⬜ Predictive memory — surface memories before they are explicitly asked for
- ⬜ Associative recall — one memory triggers a cascade of related memories
- ⬜ Temporal reasoning — understand that memories exist in time, not just space

---

## Phase 4 — Product

- ⬜ Claude API integration
- ⬜ Postgres backend for scale
- ⬜ Developer dashboard
- ⬜ API key management
- ⬜ Usage metering
- ⬜ Public launch

---

## Core Principles

1. User owns their memory — full transparency, full control
2. Intelligent forgetting protects privacy — no hoarding, no surveillance
3. Explainable — users understand why memories survive or fade
4. Cannot be weaponized — no psychological profiling, no manipulation
5. Open about how it works — formula is public, logic is clear
